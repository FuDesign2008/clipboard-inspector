# 3.2 桌面端竞品深度分析

桌面剪贴板管理器是整个剪贴板工具生态中用户量最大、竞争最充分的类别。理解这些产品的优势和局限，有助于定位Clipboard Inspector在桌面工具生态中的互补角色。

## Maccy

**基本信息**：p0deje/Maccy | 19,422 stars [^1] | MIT许可 | Swift语言 | 仅macOS

Maccy是macOS平台最受欢迎的剪贴板管理器，由Anton Sotkov开发。它的设计哲学是"做一件事并做好"：记录剪贴板历史，让你快速找到之前复制的内容。

**核心亮点**

体积只有3.2MB，这在动辄上百MB的桌面应用生态中几乎是个奇迹。Maccy启动即消失在菜单栏，不占Dock位置，不抢注意力。交互完全围绕键盘设计：`Cmd+Shift+C`呼出历史列表，输入关键词过滤，按序号选择粘贴。整个过程不需要鼠标。

v2.4版本引入的OCR搜索是一个差异化功能 [^2]。用户可以搜索复制过的图片中的文字内容，这对于截屏工作流非常实用。标签系统允许给历史条目打标记，方便长期管理。

原生macOS UI是另一个卖点。Maccy不使用Electron或其他跨平台框架，而是用Swift和AppKit构建，所以菜单、动画、快捷键都和系统风格完全一致。

**主要弱点**

仅限macOS，这是最大限制。Windows和Linux用户完全无法使用。

没有剪贴板内容检查功能。Maccy只关心"你复制了什么文本/图片"，不关心这些内容的MIME类型、数据结构或技术细节。开发者如果需要检查剪贴板里到底存了什么格式的数据，Maccy帮不上忙。

没有Markdown或ZIP导出功能。历史条目只能逐条复制出来，不支持批量结构化导出。

**定价策略**

开源版本免费，GitHub Releases直接下载。App Store版本定价$9.99 [^3]，功能相同，付费主要是支持开发者和获得自动更新。

**特性速览**

| 特性 | 支持情况 |
|------|------|
| 剪贴板历史 | 支持，默认999条 |
| 搜索 | 支持（文本+OCR） |
| 快捷键操作 | 支持，核心交互方式 |
| 文本/图片 | 均支持 |
| 代码/文件 | 支持文本，不显示文件路径 |
| MIME检查 | 不支持 |
| 导出 | 不支持 |
| 跨平台 | 仅macOS |
| iCloud同步 | 不支持 |
| 开源 | MIT许可 |

## CopyQ

**基本信息**：hluk/CopyQ | 11,594 stars [^4] | GPLv3许可 | C++语言 | 跨平台（Windows/macOS/Linux）

CopyQ是这个领域的技术天花板。如果用一个词形容它，那就是"可编程"。从2007年开始开发，至今已发布86个版本 [^5]，是剪贴板管理器中历史最悠久的项目之一。

**核心亮点**

脚本引擎是CopyQ最大的差异化优势。用户可以用内置的脚本语言（基于JavaScript语法）编写自动化流程，比如：自动将复制的URL转换为Markdown链接，自动过滤掉超过1000字符的内容，或者将特定格式的内容自动分类到不同标签页。

CopyQ能处理所有MIME类型的数据 [^6]。不同于大多数剪贴板管理器只关注纯文本和图片，CopyQ能捕获和展示HTML格式、RTF格式、自定义应用数据等。每个剪贴板条目都显示完整的MIME类型列表。

跨平台支持让它成为Windows和Linux用户的首选。尤其是在Linux桌面环境中，CopyQ几乎是唯一成熟的剪贴板管理器选择。

标签页系统允许用户按项目或类型组织历史条目。可以创建"工作"、"个人"、"代码片段"等标签页，各自维护独立的剪贴板历史。

**主要弱点**

UI是CopyQ最大的短板。社区中流传着一句评价："functional but not friendly"（功能强大但不够友好）[来源：Reddit r/macapps 多个讨论帖]。设置界面复杂，功能入口深，新手很难快速上手。

学习曲线陡峭。要充分利用脚本引擎，需要阅读大量文档。CopyQ的脚本API文档虽然完整，但组织方式偏技术手册风格，缺少渐进式教程。

macOS上的体验不如Maccy流畅。C++和Qt构建的界面在macOS上显得格格不入，字体渲染、快捷键响应、系统集成都有差距。

**定价策略**

完全免费，GPLv3开源。没有付费版本，没有商业模型。

**特性速览**

| 特性 | 支持情况 |
|------|------|
| 剪贴板历史 | 支持，无上限（可配置） |
| 搜索 | 支持（全文搜索） |
| 脚本引擎 | 支持，核心差异化 |
| MIME处理 | 支持所有类型 |
| 标签页 | 支持 |
| 跨平台 | Windows/macOS/Linux |
| 命令行 | 支持 |
| OCR | 不支持（需外部脚本） |
| 导出 | 通过脚本实现 |
| 外观 | Qt风格，偏技术向 |

## Ditto

**基本信息**：sabrogden/Ditto | 6,230 stars [^7] | GPLv3许可 | C语言 | 仅Windows

Ditto是Windows平台上历史最悠久的剪贴板管理器，已经活跃了超过15年。虽然开发节奏已经放缓，但用户基数依然庞大。

**核心亮点**

局域网同步是Ditto独有的功能 [^8]。多台Windows电脑可以在同一局域网内同步剪贴板历史，对于在多台PC之间工作的用户非常实用。同步走的是加密通道，安全性有基本保障。

轻量。安装包不到10MB，内存占用通常在20-30MB，对系统资源的消耗极低。

成熟稳定。十几年的迭代让Ditto在处理各种边缘情况时非常可靠，崩溃率极低。许多Windows用户把它列为"装机必备"。

**主要弱点**

仅限Windows。这是最大的限制，而且没有跨平台的计划。

UI过时。Ditto的界面停留在Windows XP时代的设计风格，列表视图、右键菜单、设置对话框都给人强烈的年代感。虽然功能上没问题，但视觉体验远不如现代应用。

开发缓慢。最近几年的更新频率明显降低，主要做维护性修复，很少引入新功能。社区中的功能请求积压了大量未处理的issue。

**定价策略**

完全免费，GPLv3开源。

**特性速览**

| 特性 | 支持情况 |
|------|------|
| 剪贴板历史 | 支持，可配置上限 |
| 搜索 | 支持（基础全文） |
| LAN同步 | 支持，核心差异化 |
| 文本/图片 | 均支持 |
| 快捷键 | 支持 |
| 跨平台 | 仅Windows |
| 脚本 | 不支持 |
| OCR | 不支持 |
| 导出 | 基础文本导出 |
| 外观 | 过时，Windows经典风格 |

## CleanClip

**基本信息**：macOS/iOS平台 | 商业软件（Freemium） | Swift语言

CleanClip是近两年在macOS平台崛起的剪贴板管理器，尝试用创新功能和商业模式在Maccy主导的市场中找到空间。

**核心亮点**

粘贴队列是CleanClip最独特的功能 [^9]。用户可以将多个剪贴板条目按顺序排入队列，然后逐个粘贴到目标应用。这解决了"从A复制多个内容，依次粘贴到B的多个位置"这个常见但一直没人解决的痛点。

智能列表根据内容类型自动分类：链接、图片、代码、文件等各自归入不同列表，不需要用户手动整理。

iCloud同步让macOS和iOS设备之间共享剪贴板历史，填补了Apple Universal Clipboard只同步"最新一条"的不足。

**主要弱点**

平台限制。只有macOS和iOS版本，没有Windows或Linux支持。

免费版限制严格。每日只能使用25次粘贴操作 [^10]，对于高频用户来说远远不够。这个限制把大量潜在用户推向了Maccy。

**定价策略**

采用Freemium模型。免费版有每日25次限制。付费版分为三档：标准版$19.99（一次性）、专业版$39.99（一次性）、全家桶$59.99（一次性）[来源：CleanClip官网]。这一定价在开源工具主导的市场中偏高。

**特性速览**

| 特性 | 支持情况 |
|------|------|
| 剪贴板历史 | 支持 |
| 粘贴队列 | 支持，独有功能 |
| 智能分类 | 支持 |
| iCloud同步 | 支持 |
| 跨平台 | macOS/iOS |
| 快捷键 | 支持 |
| 免费版 | 每日25次 |
| 付费版 | $19.99-$59.99 |
| 开源 | 否 |

## Raycast Clipboard History

**基本信息**：macOS平台 | 商业软件 | TypeScript/Electron | 免费/Pro $8/月

Raycast Clipboard History不是一个独立产品，而是Raycast启动器的一个内置扩展。但这种"不独立"恰恰是它的优势：对于已经使用Raycast的用户来说，剪贴板历史功能是零成本获得的。

**核心亮点**

零额外安装。Raycast用户只需在扩展商店中启用Clipboard History，不需要下载任何新应用。Raycast在macOS开发者社区的渗透率已经非常高 [^11]，这意味着大量目标用户已经具备使用条件。

密码排除是一个实用的安全功能。Raycast会自动检测来自1Password、Bitwarden等密码管理器的内容，将其排除在剪贴板历史之外，避免敏感信息被记录 [来源：Raycast官方文档]。

云同步是Pro版的功能，剪贴板历史可以通过Raycast的云服务在多台Mac之间同步。

**主要弱点**

必须使用Raycast。这是最大的前提条件。对于不用Raycast的用户，这个功能完全不存在。

32,768字符限制 [^12]。单条剪贴板内容超过这个长度就会被截断。对于复制大段代码或日志的开发者来说，这个限制会丢掉关键信息。

**定价策略**

剪贴板历史功能包含在免费版中。云同步和其他高级功能需要Raycast Pro，定价$8/月或$96/年 [来源：Raycast官网]。

**特性速览**

| 特性 | 支持情况 |
|------|------|
| 剪贴板历史 | 支持 |
| 搜索 | 支持（Raycast全局搜索） |
| 密码排除 | 支持 |
| 云同步 | Pro版 |
| 独立使用 | 否（需Raycast） |
| 字符限制 | 32,768字符 |
| 跨平台 | 仅macOS |
| 快捷键 | Raycast全局快捷键 |
| 免费 | 基础功能免费 |

## 桌面端竞品总结

| 维度 | Maccy | CopyQ | Ditto | CleanClip | Raycast |
|------|------|------|------|------|------|
| 用户规模 | 最大（macOS） | 中等（跨平台） | 大（Windows） | 小（增长中） | 大（Raycast用户） |
| 技术深度 | 低 | 最高 | 低 | 中 | 低 |
| 创新性 | 中（OCR） | 中（脚本） | 低（LAN） | 高（粘贴队列） | 低（集成） |
| 维护活跃度 | 高 | 高 | 低 | 高 | 高 |
| 定价 | 免费/$9.99 | 免费 | 免费 | Freemium | 免费/$8月 |
| 剪贴板检查 | 无 | 部分MIME | 无 | 无 | 无 |

关键发现：没有任何桌面端竞品提供真正的"剪贴板内容检查"功能。CopyQ能看到MIME类型列表，但缺少结构化展示和导出能力。这个空白正是Clipboard Inspector可以填补的。
